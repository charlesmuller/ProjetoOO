Um sistema de cadastro de bugs e roteiros de testes, voltadas a área da qualidade de sistemas. 
Será possível cadastrar no roteiro de teste seus casos de teste (irá conter título, objetivo, passos para realização, resultado esperado e status);
Poderá ser cadastrado bugs, defeitos, falhas e melhorias, contendo um breve resumo, uma descrição, um passo-a-passo, gravidade (pequeno,médio e grande), 
uma prioridade (baixa, média e alta) e um [Caso de Teste] que foi realizado e encontrado esse bug.

Descrição:

Você foi convidado a participar de um Hackaton em Java. Para isso, algumas regras foram estabelecidas:
- Podem ser desenvolvidos trabalhos em duplas ou trios
- O assunto é livre, desde que seja desenvolvido um Sistema Computacional, utilizando o Paradigma de Orientação a Objetos e a Linguagem Java e observados os seguintes itens:
   - Encapsulamento de classes (métodos acessores e modificadores para todos os atributos)
   - Instanciação de objetos (métodos construtores com e sem parâmetros)
   - Relacionamento entre objetos 
   - Herança
   - Polimorfismo
   - Conjuntos de objetos (Array/ArrayList/Set/Map)
  
Este trabalho deve ser realizado observando etapas, a saber:
Etapa 1: definição da equipe e assunto do projeto - 20/10 (semipresencial 4) 
Etapa 2: apresentação da modelagem (diagrama de classes) - 10/11 
Etapa 3: entrega do projeto (código fonte) - 01/12 
Etapa 4: entrega da documentação do projeto - 01/12
